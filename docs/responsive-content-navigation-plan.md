# Plan de pulido responsive y navegación de contenido

## Commit sugerido

```text
web(ui): polish responsive layout and content navigation
```

## Objetivo

Pulir la experiencia visual y de navegación del portafolio, manteniendo coherencia editorial, trazabilidad e internacionalización progresiva.

## Validación por pantalla

- 360 px: móvil pequeño.
- 390/430 px: móvil estándar.
- 768 px: tablet.
- 1024/1280 px: laptop.
- 1440 px o superior: desktop amplio.

## Checklist

- Header no se rompe.
- Foto y marca mantienen proporción.
- Selector de idioma no desplaza el menú.
- Modo claro/oscuro mantiene contraste.
- Cards no quedan desalineadas.
- CV mantiene botones claros.
- Publicaciones tienen detalle y resumen.
- Software distingue registro, prototipo y herramienta.
- QR apunta a `/cv/`.
- English y Nederlands quedan como idiomas planificados hasta traducción validada.
